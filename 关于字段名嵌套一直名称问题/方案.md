## 问题分析

**核心现象**：每次通过高级筛选节点配置（或其他含字段选择的节点）操作后，字段名在列表框/下拉框中视觉上多占一行，随使用次数累积导致列表越来越长。

---

### 根因分析

代码审查定位到 **两处相互叠加的问题**：

**问题1：`refresh_combo_values` 的 `keep_custom`逻辑导致「幽灵条目」残留**

在 `build_filter_config`（以及其他节点配置）中，刷新字段来源时调用：

```python
self.refresh_combo_values(field_combo, field_var, all_values, keep_custom=True,...)
```

`refresh_combo_values` 的逻辑是：若当前值不在新列表中，就**把当前值插到列表最前面**，同时**不重置**变量。结果是：

1. 用户选了`"表A.字段1"` → 配置保存
2. 用户改变副表选择，刷新后 `"表A.字段1"` 不再有效
3. `keep_custom=True` → `"表A.字段1"` 被插到下拉列表顶部，但变量不被重置
4. 下次刷新：`field_var` 仍是旧值，又在顶部插一次……且此逻辑在条件选择/匹配字段三个combo 上同时发生

每次「保存表选择/刷新字段」都可能在三个 combo 的值列表前多插一个失效条目，用户在选字段时看到的候选列表越来越长。

**问题2：外部数据（Word/Excel 读取）列名中可能含`\n`，传入 `get_plan_filter_available_fields` 后未清洗**

`get_plan_filter_available_fields` 构建字段名：

```python
fields.append(f"{table}.{col}")
```

若 `col` 来自 Word/Excel 读取结果（`word_excel_read_to_db_plugin_v1.py`）且原始单元格含换行符，`col` 就带有 `\n`。Tkinter 的 Listbox 对含 `\n` 的条目会渲染成两行（原文字一行 + 空一行），即「嵌套一行」，且每次刷新都会重现，行为与节点来源数据挂钩。

---

### 更改方案（不改代码，仅描述修改点）

**修改一：`refresh_combo_values`，去掉或收紧 `keep_custom` 行为**

位置：`DataFlowKit.py`，`refresh_combo_values` 方法（约第 11152 行）

方案：当 `keep_custom=True` 且当前值不在新列表时，**不再插入当前值**，而是直接用 `fallback` 重置变量。可新增一个参数 `keep_custom=False` 作为更安全的默认值，或者在 `build_filter_config` 的几处调用中改传`keep_custom=False`——因为筛选字段的 combo 本来就不应该保留已经不再合法的字段值。

具体：将 `build_filter_config` 内对 `field_combo`、`left_combo`、`right_combo`、`value_combo` 的四处`refresh_combo_values` 调用改为 `keep_custom=False`。

**修改二：`get_plan_filter_available_fields`，对字段名 strip**

位置：`DataFlowKit.py`，`get_plan_filter_available_fields` 方法（约第 18949 行）

方案：在拼接字段名前先清洗 `col`，去掉首尾空白和换行：

```python
# 原来
fields.append(f"{table}.{col}")
# 改为
col_clean = str(col or "").strip().replace("\r", "").replace("\n", " ")
fields.append(f"{table}.{col_clean}")
```

同样地，`current_fields` 的构建：

```python
# 原来
fields = [f"当前表.{h}" for h in headers]
# 改为
fields = [f"当前表.{str(h or '').strip()}" for h in headers]
```

**修改三：`refresh_listbox_values`，避免 `selection_set` 触发 `<<ListboxSelect>>` 事件**

位置：`DataFlowKit.py`，`refresh_listbox_values` 方法（约第 11164 行）

方案：在刷新前临时解绑 `<<ListboxSelect>>` 绑定，批量填充完毕后再重新绑定，防止每次 `selection_set` 都触发 `sync_output_fields`（或类似回调）写入未完成状态的`config["output_fields"]`。

### 优先级建议

| 优先级 | 修改点 | 效果 |
|---|---|---|
| 高 | `refresh_combo_values` 调用改`keep_custom=False` | 直接消除 combo 条目累积 |
| 高 | `get_plan_filter_available_fields` 的 `strip()` | 消除换行字符导致的视觉双行 |
| 中 | `refresh_listbox_values` 临时解绑事件 | 避免中间状态被写入 config |

三处改动代码量很少，不涉及功能逻辑改变，均属防御性数据清洗和事件控制。
